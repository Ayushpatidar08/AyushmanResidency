# Vercel Hosting Guide

This guide explains step-by-step how to prepare and host your project on Vercel, designed for someone with zero prior hosting experience.

## 1. Project Folder Cleanup (What to Keep vs. Remove)

Before hosting, you must ensure your project is clean. You do this by pushing your code to GitHub (which Vercel will read). You should **never** manually upload certain files.

### Files to KEEP (Upload to GitHub)
*   **`src/` folder:** Contains your actual code (React components, styles, etc.).
*   **`public/` folder:** Contains static files like images and your `index.html`.
*   **`package.json` & `package-lock.json` (or `npm-shrinkwrap.json` / `yarn.lock`):** Extremely important. These tell Vercel exactly which packages to install.
*   **`.gitignore`:** Tells Git which files to ignore (see below).
*   **Configuration files:** `vite.config.ts`, `tsconfig.json`, `tailwind.config.js`, `postcss.config.js`.

### Files to REMOVE / DO NOT UPLOAD (Ensure they are in your `.gitignore`)
*   **`node_modules/` folder:** **NEVER upload this.** It contains thousands of downloaded packages. It's huge. Vercel will look at your `package.json` and download these automatically on their own super-fast servers.
*   **`.env` file:** **NEVER upload this.** This file contains your private API keys or database passwords. If you upload it, hackers can steal your keys. You will enter these securely in the Vercel dashboard later.
*   **`dist/`, `build/`, or `.next/` folders:** These are compiled files. Vercel automatically builds the project itself, so uploading them is redundant and can cause conflicts.
*   **`.DS_Store` or `Thumbs.db`:** Useless system files generated by Windows/Mac.

## 2. Step-by-Step Hosting Guide

### Step A: Push Your Code to GitHub
Vercel works best when connected to a GitHub repository.
1.  Go to [github.com](https://github.com) and create an account if you don't have one.
2.  Create a "New Repository" (a folder on GitHub for your project).
3.  Follow the instructions on GitHub to push your local code (from VS Code) to this new repository. Ensure you do not commit `node_modules` or `.env`.

### Step B: Create a Vercel Account
1.  Go to [vercel.com](https://vercel.com).
2.  Click **Sign Up** and choose **Continue with GitHub**. This links your Vercel account directly to your GitHub code.

### Step C: Import Your Project
1.  Once logged into Vercel, click the **Add New...** button and select **Project**.
2.  You will see a list of your GitHub repositories. Find the one you just created and click **Import**.

### Step D: Configure the Project
1.  **Project Name:** You can leave the default.
2.  **Framework Preset:** Vercel usually auto-detects this (e.g., Vite, Next.js, Create React App). Leave it as is if it detects it correctly.
3.  **Root Directory:** Leave it as `./` (unless your project is inside a subfolder).
4.  **Build and Output Settings:** Leave default. Vercel knows the standard commands (like `npm run build`).
5.  **Environment Variables (Crucial Step):**
    *   Remember that `.env` file we did not upload? This is where you add it.
    *   Open your local `.env` file.
    *   Copy each key-value pair and paste them into Vercel's Environment Variables section one by one.
    *   *Example:* If your local `.env` has `VITE_API_KEY=12345`, put `VITE_API_KEY` in the Name box, and `12345` in the Value box. Click **Add**.

### Step E: Deploy
1.  Click the large **Deploy** button.
2.  Vercel will now take over. You will see a screen showing building progress:
    *   It downloads a fresh operating system container.
    *   It runs `npm install` (creating a fresh `node_modules` folder).
    *   It runs `npm run build` (creating the final optimized website).
3.  If everything is correct, you will see confetti and a success screen!

### Step F: Visit Your Live Site
*   Vercel will provide a free, secure HTTPS link (e.g., `your-project-name.vercel.app`).
*   Click the URL or the website preview image to see your live site on the internet. Any future changes you push to GitHub will now automatically update this live site.
